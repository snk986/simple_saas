# Roadmap

This roadmap focuses on making Calyra AI easier to self-host, extend, and
maintain as open-source software.

## v0.2 Open Source Hardening

- Add repeatable Supabase setup and seed documentation.
- Add contract tests for audio provider adapters.
- Add request/response fixtures for webhook finalization flows.
- Document provider capability differences: latency, callbacks, output formats,
  and cost considerations.
- Improve security docs for service-role usage, RLS, and private report routes.

## v0.3 Provider Ecosystem

- Add a provider adapter checklist.
- Add examples for swapping AI text providers.
- Add examples for swapping audio generation providers.
- Add fallback handling documentation for provider outages.

## v0.4 Contributor Experience

- Add more good-first-issue tasks.
- Add architecture decision records for provider contracts and credits.
- Add local development fixtures for public song pages.
- Add deployment recipes beyond Vercel.

## Long-Term Direction

- Make the generation workflow more modular.
- Improve accessibility and keyboard support.
- Improve i18n coverage and translation contribution docs.
- Add broader test coverage for auth, credits, storage, and webhooks.

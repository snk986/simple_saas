# Calyra AI 专业歌曲制作流程

## 1. 文档目的

本流程用于稳定制作可用于产品展示、内容营销和质量标杆的高质量歌曲。

核心原则：

- 不接受模型未经编辑的第一版结果。
- 歌词、制作风格和最终音频必须分别经过质量检查。
- 不复制具体歌曲、歌词、旋律或艺人的可识别风格。
- 用户作品未经明确授权，不得公开展示或用于营销。
- 音频生成成功是主流程；封面、报告等增强功能失败不得阻断歌曲完成。

## 2. 整体流程

```text
确定创作目标：模仿欧美排行榜上前50的热歌
→ DeepSeek 生成歌词与 Style 初稿
→ GPT 深度改写歌词与 Style
→ 独立 GPT 审核与微调
→ 人工终审
→ Calyra 提交 fal 生成音频
→ 音频质量检查与版本选择
→ 封面、案例内容与发布授权
```

每个阶段由不同角色负责：

| 阶段 | 角色 | 主要职责 |
| --- | --- | --- |
| 创作简报 | 创意负责人 | 定义听众、主题、情绪、曲风和传播目标 | 模仿欧美排行榜上前50的热歌
| 初稿生成 | DeepSeek | 快速产出完整歌词结构和详细制作 Style |
| 深度改写 | GPT 编辑窗口 | 提升 Hook、可唱性、意象统一性和情绪推进 |
| 独立审核 | GPT 审核窗口 | 从零审查问题，提出小范围精准调整 |
| 人工终审 | 制作人 | 决定是否达到音频生成标准 |
| 音频生成 | Calyra + fal | 根据最终歌词和 Style 生成候选音频 |
| 成品质检 | 制作人 | 选择最佳版本，检查演唱、结构和音频质量 |

## 3. 阶段一：创作简报

在生成歌词前，先建立一份简短但明确的创作简报，模仿欧美排行榜上前50的热歌。

至少定义：

- 目标听众
- 核心情绪
- 故事或主题
- 歌曲类型
- 演唱角色
- 期望使用场景
- 最重要的一句 Hook

示例：

```text
目标听众：喜欢现代 R&B 和 melodic hip-hop 的年轻听众
核心情绪：在冰冷城市中找到温暖的爱情
演唱角色：男声旋律说唱主歌，女声 R&B 副歌，最终合唱
歌曲气质：亲密、浪漫、现代、带一点怀旧
传播目标：副歌适合制作 20 秒短视频
核心 Hook：If the night turned neon, I'd light it up for you
```

## 4. 阶段二：DeepSeek 生成初稿

### 4.1 目标

快速生成一份结构完整、方向明确的歌词和 Style 初稿，为后续编辑提供素材。

### 4.2 歌词要求

初稿必须包含明确段落：

```text
[Intro]
[Verse 1]
[Pre-Chorus]
[Chorus]
[Verse 2]
[Chorus]
[Bridge]
[Final Chorus / Outro]
```

根据歌曲需要，可标注演唱与编曲方向：

```text
[Verse 1 - Melodic male rap-sung lead]
[Chorus - Smooth female R&B hook]
[Bridge - Stripped, intimate duet]
```

### 4.3 Style 要求

Style 不能只写宽泛类型，例如 `Pop` 或 `R&B`。应尽量包含：

- 主曲风与辅助曲风
- BPM 或速度感
- 调性或明暗倾向
- 主唱类型与演唱方式
- 乐器与音色
- 鼓点和低频特征
- 编曲动态
- 混音质感
- 整体场景和情绪

示例：

```text
Modern R&B and melodic hip-hop soul duet, 69 BPM halftime feel,
soft melodic male rap-sung verses with a smooth female R&B hook,
warm vintage soul-sample texture, lush romantic strings, rounded bass,
restrained trap-influenced drums, intimate glossy mix,
nostalgic but modern city-night love atmosphere.
```

### 4.4 创作边界

可以分析流行歌曲常见的结构、情绪推进和制作规律，但不得：

- 复现现有歌曲歌词或独特表达
- 要求生成特定歌曲的近似版本
- 模仿可明确识别的在世艺人声音或个人风格
- 复制现有旋律、Hook 或标志性编曲

目标是学习类型规律，不是复制作品。

## 5. 阶段三：GPT 深度改写

### 5.1 目标

将初稿从“完整”提升到“有记忆点、可演唱、有统一世界观”。

### 5.2 歌词检查项

重点检查并改写：

- 标题是否与核心 Hook 强关联
- 副歌是否简短、明确、容易重复
- 每段歌词是否推动情绪或故事
- 意象是否属于同一个世界
- 句子长度是否适合演唱
- 押韵是否自然，而不是为了押韵牺牲表达
- 是否存在空泛、陈词滥调或 AI 感明显的句子
- Verse、Chorus 和 Bridge 是否有明显功能差异
- Final Chorus 是否提供足够的情绪释放

### 5.3 Style 检查项

重点检查并改写：

- Style 是否与歌词情绪一致
- 人声安排是否与歌词角色对应
- BPM、鼓点和段落结构是否协调
- 乐器是否过多或互相冲突
- 是否明确描述主歌、副歌、Bridge 的动态变化
- 是否能让音乐模型理解最重要的制作方向

### 5.4 改写原则

- 保留最强的原始创意。
- 优先改写弱句，不为了变化重写所有内容。
- 先解决 Hook、结构和可唱性，再处理局部修辞。
- Style 应精确但不过度堆砌形容词。

## 6. 阶段四：独立 GPT 审核与微调

### 6.1 独立审核要求

审核必须在独立窗口或独立上下文中进行，避免审核者受到前一轮创作过程影响。

审核者不负责大规模重写，而是：

- 找出最明显的三个问题
- 判断歌词和 Style 是否互相支持
- 判断是否达到音频生成标准
- 仅对必要位置进行微调

### 6.2 审核评分

每项按 10 分制评分：

| 项目 | 合格线 |
| --- | --- |
| Hook 记忆度 | 8 |
| 歌词可唱性 | 8 |
| 结构完整度 | 8 |
| 意象统一性 | 8 |
| 情绪推进 | 8 |
| Style 清晰度 | 8 |
| 歌词与 Style 匹配度 | 8 |
| 原创性与版权安全 | 必须通过 |

任一核心项低于 8 分，应返回深度改写阶段。

### 6.3 审核输出

审核窗口应输出：

```text
结论：通过 / 需要微调 / 需要重写

最强部分：
1.
2.

主要问题：
1.
2.
3.

建议调整：
1.
2.

最终歌词：
...

最终 Style：
...
```

## 7. 阶段五：人工终审

提交音频生成前，由制作人完成最终检查：

- Hook 是否能在第一次听后被记住
- 歌词中是否存在尴尬、歧义或不自然表达
- Style 是否足够明确且没有互相冲突的要求
- 是否适合目标受众和传播场景
- 是否存在版权、品牌或内容安全风险

通过后，将最终歌词和 Style 锁定，不再随意修改。

## 8. 阶段六：音频生成

使用 Calyra 将最终歌词和 Style 提交给 fal 音乐模型。

要求：

- 每次生成保存 provider task id。
- 用户离开页面后，歌曲仍应由 webhook 或 cron 完成。
- 同一任务的 poll、webhook 和 cron 必须共用 server-side finalizer。
- 音频生成成功后立即将歌曲标记为 `ready`。
- 封面生成失败不得阻断歌曲完成。
- fal 未提供封面时，使用 Calyra 品牌默认封面。

建议对重要展示歌曲生成至少两个候选版本。

## 9. 阶段七：音频质量检查

分别完整试听每个候选版本，并检查：

- 主唱音色是否符合 Style
- 男女声或角色切换是否自然
- 歌词发音是否正确
- 副歌是否真正形成高潮
- 编曲是否有清晰的动态变化
- Bridge 和 Final Chorus 是否有效
- 是否出现异常噪声、断句、吞字或结构错误
- 成品是否适合截取 15 至 30 秒宣传片段

候选版本评分：

| 项目 | 权重 |
| --- | --- |
| 第一遍吸引力 | 20% |
| Hook 表现 | 25% |
| 人声表现 | 20% |
| 编曲与动态 | 15% |
| 歌词表达 | 10% |
| 音频完成度 | 10% |

选择综合分最高的版本作为主版本。

## 10. 阶段八：展示与宣传

### 10.1 自有演示歌曲

对于团队自己制作的歌曲，可以：

- 加入首页精选歌曲
- 制作歌词到成品的案例页
- 发布 15 至 30 秒高潮片段
- 展示创作简报、歌词结构和 Style 的演进过程

### 10.2 用户歌曲

用户歌曲默认保持私有。用于公开展示或营销前必须获得明确授权，并确认：

- 用户同意公开展示
- 用户选择展示的版本
- 用户希望实名、昵称还是匿名
- 是否允许用于社交媒体短视频
- 是否允许展示部分歌词和创作过程

未获得明确回复，不得公开或用于宣传。

## 11. 《Neon World》案例总结

《Neon World》的高质量主要来自：

- 具体且可执行的制作 Style
- 明确的男女声角色安排
- 完整的歌曲段落和动态标注
- 简短且高度统一的核心 Hook
- 持续使用城市夜景意象
- 主歌、Bridge 和 Final Chorus 之间有明显变化
- 初稿、深度改写和独立审核组成的多轮编辑流程

该案例说明：

> 高质量 AI 歌曲的核心不是一次生成，而是专业创作流程、清晰制作指令和多轮质量控制。

## 12. 可产品化方向

未来可以逐步将内部流程产品化：

1. 用户提交故事、情绪或歌词。
2. 系统生成结构化歌词初稿。
3. 系统进行专业改写。
4. 独立审核模型评分。
5. 低于质量阈值时自动微调一次。
6. 输出最终歌词和详细 Style。
7. 用户确认后生成音频。
8. 音频完成后生成评判报告和分享素材。

产品对外可以描述为：

```text
Calyra 不会停在第一版歌词。
每首精选歌曲都会经过结构化创作、专业改写和独立质量审核，
再进入音乐生成阶段。
```
